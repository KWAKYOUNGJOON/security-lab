# Web Assessment Checklist

## 정보수집

- 대상 도메인, 서브도메인, IP, 포트 범위를 식별했는가
- 기술 스택, WAF, 프레임워크, 인증 방식, 외부 연동 지점을 파악했는가
- 관리자 페이지, API, 파일 업로드, 공개 저장소 노출 여부를 확인했는가

## 인증/인가

- 기본 계정, 약한 비밀번호, 계정 열거가 가능한가
- 수평 권한 상승과 수직 권한 상승이 가능한가
- 비인가 사용자가 직접 객체 참조로 타 사용자 데이터에 접근 가능한가
- 관리자 기능 호출에 추가 검증이 누락되어 있는가

## 세션

- 세션 쿠키에 `HttpOnly`, `Secure`, `SameSite`가 적절히 설정되어 있는가
- 로그인 후 세션 고정이 가능한가
- 로그아웃 후 세션 재사용이 가능한가
- 장시간 미사용 세션 만료 정책이 적절한가

## 입력검증

- SQL Injection, NoSQL Injection, Command Injection 가능성을 점검했는가
- XSS, SSTI, 템플릿 주입, 경로 조작을 점검했는가
- 파일 업로드 확장자, MIME, 저장 위치 검증이 적절한가
- 서버와 클라이언트 검증이 불일치하는 입력 지점이 있는가

## 클라이언트

- DOM 기반 XSS 가능성이 있는가
- 민감정보가 프런트엔드 코드, source map, local storage에 남는가
- CSP, iframe 제어, CORS 정책이 적절한가
- 숨김 기능이나 비활성화된 엔드포인트가 노출되어 있는가

## 로깅/예외처리

- 에러 메시지에 스택트레이스, 경로, 계정 정보가 노출되는가
- 보안 이벤트가 감사 로그에 남는가
- 인증 실패, 권한 실패, 입력 검증 실패 로그가 구분되는가
- 로그에 토큰, 세션, 개인정보가 평문으로 남는가
