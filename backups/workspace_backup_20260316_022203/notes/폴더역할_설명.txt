취약점 진단 폴더 구조 및 역할 설명
작성일: 2026-03-15
기준 루트: D:\취약점 진단

[폴더별 역할]

1. hexstrike-ai
- 자동 웹 취약점 진단 도구 본체를 보관하는 폴더
- HexStrike AI 관련 소스코드, 설정, 실행 스크립트를 관리
- 스캔 실행 자체는 이 폴더를 기준으로 진행

2. outputs
- 실제 진단 결과 원본을 저장하는 폴더
- 진단 1회마다 run 단위 폴더를 생성하여 결과를 분리 보관
- 예시:
  D:\취약점 진단\outputs\run_2026-03-15_001
- 내부 권장 파일명:
  nuclei.jsonl
  ffuf.json
  httpx.jsonl
  nikto.json
  burp.xml
  screenshots\
  notes.txt

3. 보고서
- 보고서 생성 코드와 결과물을 관리하는 폴더
- 내부의 reporting_starter_v3_package는 보고서 엔진 코드
- artifacts 폴더는 생성된 보고서 결과물 저장 위치로 사용 권장
- 예시:
  D:\취약점 진단\보고서\artifacts\run_2026-03-15_001

4. real_fixtures
- 실제 도구 출력 샘플 원본을 모아두는 폴더
- parser 수정, 테스트, 재현 검증 시 사용
- 민감정보는 반드시 제거 또는 익명화 후 보관

5. scripts
- 운영용 배치파일과 실행 보조 스크립트를 저장하는 폴더
- 예시:
  run_report.bat
  run_tests.bat
  open_latest_report.bat

6. notes
- 운영 메모, 규칙, 수동 점검 내용, TODO를 기록하는 폴더
- 사람이 읽는 설명 문서는 이 폴더에 보관
- 현재 파일도 notes 폴더에 두는 것을 권장

[운영 원칙]

1. 진단 결과는 무조건 outputs 아래에 저장한다.
2. 보고서 엔진은 outputs의 run 폴더를 직접 읽는다.
3. 결과 파일은 복사/붙여넣기하지 않는다.
4. 보고서는 보고서\artifacts 아래에 생성한다.
5. 같은 run 이름을 결과 원본과 보고서 생성물에 동일하게 사용한다.

[run 폴더 이름 규칙]

- 형식:
  run_YYYY-MM-DD_번호
- 예시:
  run_2026-03-15_001
  run_2026-03-15_002

[권장 작업 순서]

1. hexstrike-ai에서 진단 실행
2. 결과를 outputs\run_xxx 폴더에 저장
3. 보고서 엔진에서 pipeline.py 실행
4. report.json, report.md, report.docx 생성 확인
5. 필요 시 수동 검토 후 최종본 확정

[메모]

- 폴더 이름은 짧고 고정된 이름을 유지한다.
- 역할 설명은 폴더명에 쓰지 말고 notes 문서에 적는다.
- 원본 결과와 보고서 결과는 반드시 분리한다.
- fixtures에는 실제 테스트에 필요한 샘플만 정리한다.
