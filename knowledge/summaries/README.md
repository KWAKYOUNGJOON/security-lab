# Summaries Guide

`reference`의 원본 자료를 바로 수정하지 말고, 필요한 핵심 내용만 `knowledge`로 요약해 옮긴다.

## 요약 규칙

1. 원본 파일명과 출처 폴더를 함께 기록한다.
2. 원문 복사보다 핵심 기준, 점검 포인트, 예외 조건을 정리한다.
3. 버전이나 발행 연도가 중요하면 반드시 함께 적는다.
4. 원본의 표나 분류 체계는 필요한 범위만 재구성한다.
5. 실무용 체크리스트, 매핑, 프롬프트로 재활용 가능한 형태를 우선한다.

## 권장 형식

- 출처
- 적용 범위
- 핵심 요구사항
- 점검 시 주의사항
- 후속 액션
